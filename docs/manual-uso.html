DevUtils - Manual de uso

Guía rápida para configurar proyectos, ejecutar análisis en Sonar/Semgrep/Gitleaks, usar diff de ramas y generar reportes.

Nota: el uso de diff es opcional en Sonar y Semgrep.

1) Arranque del sistema

Levantar Docker
  1. Antes de ejecutar cualquier comando, configurar en docker/docker-compose.yml la ruta base del volumen HOST_ROOT (debe apuntar al directorio que contiene los proyectos a analizar).
    Ejemplo (si www es la carpeta que contiene los proyectos): host_root: &host_root ${HOST_ROOT:-C:/www}
  2. Una vez configurada la ruta, desde la raíz del proyecto ejecutar: docker compose -f docker/docker-compose.yml up -d --force-recreate.
  3. Verificar contenedores: dev_utils, sonarqube, semgrep, gitleaks.
  4. Abrir http://localhost:3000.

2) Configuración principal (crear proyectos)

Ruta: menú superior → ícono de engranaje → /app_config.

  1. Ir a Proyectos y presionar Crear proyecto.
  2. Completar:
    • nombre proyecto: solo letras y números.
    • directorio proyecto: ruta relativa a workspaceBaseDir (se puede elegir con el explorador).
  3. Guardar. El proyecto queda disponible para Sonar, Semgrep y Gitleaks.

Importante: si el directorio no existe o queda fuera del workspace, el escaneo fallará.

3) Configuraciones globales de módulos

Sonar configuración

Ruta: /sonar_config.

  • Ingresar sonarToken.
  • Guardar.
  • sonarHostUrl viene de .env (en UI se usa localhost:9000).

Semgrep configuración

Ruta: /semgrep_config.

  • Editar reglas YAML personalizadas.
  • Opcional: restaurar reglas por defecto.
  • Guardar configuración.

4) Flujo por módulo

4.1 Sonar

  1. Entrar a /sonar.
  2. Seleccionar proyecto en el combo.
  3. (Opcional) usar diff:
    • Click en diff.
    • Elegir rama base (Base) y comparar contra rama actual (Current).
    • Confirmar para cargar archivos cambiados en SOURCES.
  4. Ajustar SOURCES y EXCLUSIONS si aplica.
  5. Ejecutar SonarScanner.
  6. Al terminar, usar Descargar PDF para el reporte.
  7. Opcional: botón SonarQube abre dashboard del proyecto.

4.2 Semgrep

  1. Entrar a /semgrep.
  2. Seleccionar proyecto.
  3. (Opcional) usar diff para llenar SOURCES con cambios entre ramas.
  4. Configurar:
    • SOURCES / EXCLUSIONS.
    • Conjuntos de reglas (auto, security-audit, secrets, etc.).
    • Checkbox de archivo de reglas personalizadas.
  5. Ejecutar SemgrepScanner.
  6. Al finalizar, descargar reporte con Descargar PDF.

4.3 Gitleaks

  1. Entrar a /gitleaks.
  2. Seleccionar proyecto.
  3. Elegir modo:
    • Git (si el directorio es repo git válido).
    • Dir (escaneo de archivos del directorio).
  4. Opcional: activar/desactivar exclusión de .gitignore.
  5. Ejecutar Gitleaks.
  6. Generar reporte con Descargar PDF.

5) Ejemplo rápido (flujo recomendado)

  1. Crear proyecto en /app_config (nombre + directorio).
  2. Configurar token Sonar en /sonar_config.
  3. (Opcional) ajustar reglas en /semgrep_config.
  4. Ir a /sonar, seleccionar proyecto, (opcional) usar diff, correr scanner y descargar PDF.
  5. Ir a /semgrep, seleccionar proyecto, (opcional) usar diff, correr scanner y descargar PDF.
  6. Ir a /gitleaks, seleccionar proyecto, correr escaneo y descargar PDF.

Resultado: un ciclo completo de análisis y reporte por proyecto.

6) Problemas comunes

  • No aparecen proyectos: revisar que existan en /app_config.
  • Error en diff: confirmar que el directorio sea repositorio Git y tenga ramas válidas.
  • Sonar sin conexión: validar sonarHostUrl, token y estado de SonarQube.
  • Semgrep/Gitleaks no ejecutan: validar contenedores y acceso Docker desde la app.
